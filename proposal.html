Project Proposal

Summary

For our final project, we decided that we want to add light field camera functionality into project 3. By doing so, we would be taking in far more information per pixel, like the radiance coming in from different directions. Once we have implemented it completely, we should be able to move around in the scene and refocus within the scene, even after rendering the initial image.

Problem Description

Currently, in normal photographs, once a photo is taken, it's stuck there forever and you're not able to see any more in the scene besides what's displayed. While this is great, sometimes it would be nice to see things from a different point of view or to refocus a bad picture on a better location. By implementing a light field camera, we would be able to do all of these things.

In order to implement the light field camera, we will have to change the way each pixel takes in light. For each pixel, instead of taking in a single radiance from a single direction, we will be recording the radiance from many different incoming directions. By doing so, we can then easily slice the information that we've collected in ways to change the aperture, focal length, point of view (where the camera's located), and many other fields.

Goals and Deliverables

For this project, we plan on being able to deliver a fully functional light field camera. What this entails is that our images will have all of the data a light field camera would have recorded, aka the 4D light field. From there, we plan on being able to implement a way to be able to take any image rendered in this fashion, and be able to change where the focus is within the image. To break up the deliverables up further, we need to be able to 1) be able to collect all of the data properly for every single pixel (collect all the radiances from all directions properly), 2) transform and slice the data correctly, then 3) render the new 2D image. Creating the ability to change the image's focus location would then be a matter of slicing the data correctly so that we can render the correct image. While all of these things seem very doable, we're worried about how long it will take to get the functions up and running, since debugging them seem difficult until we can see the images, which would be much further down the pipeline.

Asssuming we can actually figure out everything ahead of schedule, we hope to implement a fully functional GUI that allows us to not only refocus the image wherever we like, but also be able to move around in the image, so that we can almost walk side-to-side, or even into the image. If we have the opportunity to do so, we would also like to try making more complex images with more data, so that we could create incredibly immersive images, where we could show off many different focal points/as well as many different points of view.
